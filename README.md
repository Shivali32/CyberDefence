## Labs

### Lab 1: Environment Setup and Linux Fundamentals
- Set up the virtual machine environment for cybersecurity labs.
- Practiced essential Linux commands and system navigation.
- Introduced file permissions, process management, and basic networking utilities.
- Established readiness for hands-on security experimentation.

### Lab 2: Cryptography and Encryption Basics
- Explored symmetric and asymmetric cryptography through practical exercises.
- Used common algorithms to understand key management and block modes.
- Analyzed common cryptographic pitfalls and misuse scenarios.
- Demonstrated how improper cryptographic usage weakens system security.

### Lab 3: Secret-Key Encryption and Modes of Operation
- Examined block cipher modes such as ECB and CBC.
- Performed hands-on experiments to observe security implications.
- Used visual and data-based analysis to identify pattern leakage.
- Reinforced best practices for secure encryption configurations.

### Lab 4: Hash Functions and Password Security
- Studied cryptographic hash functions for password storage and integrity.
- Compared weak hashing algorithms such as MD5 with stronger alternatives.
- Demonstrated vulnerabilities of fast hashing methods.
- Emphasized salting and iterative hashing as industry best practices.

### Lab 5: Public-Key Cryptography and Digital Signatures
- Implemented RSA encryption and key generation.
- Analyzed digital signatures for authentication and non-repudiation.
- Demonstrated confidentiality and integrity guarantees.
- Explored real-world applications such as secure communication and software verification.

### Lab 6: Mobile Application Reverse Engineering
- Performed static analysis of a mobile application.
- Inspected application files and configuration data.
- Identified insecure storage of sensitive information.
- Highlighted secure storage practices for mobile applications.

### Lab 7: Authentication and Password Vulnerabilities
- Analyzed common authentication weaknesses.
- Evaluated poor password policies and insecure security questions.
- Demonstrated how personal information can bypass authentication mechanisms.
- Reinforced principles of strong authentication design.

### Lab 8: Intrusion Detection Systems and Snort
- Configured and tested Snort as a network intrusion detection system.
- Developed custom Snort rules for attack detection.
- Detected port scanning and unauthorized access attempts.
- Demonstrated the role of IDS tools in network defense.

### Lab 9: Web Security and Cross-Site Scripting (XSS)
- Explored XSS vulnerabilities through hands-on exploitation.
- Demonstrated different types of XSS attacks.
- Analyzed how malicious scripts compromise user data.
- Highlighted mitigation techniques such as input validation and CSP.

### Lab 10: Steganography and Hidden Data Analysis
- Investigated techniques for hiding data in images and audio files.
- Used forensic tools to extract concealed information.
- Analyzed covert communication techniques.
- Improved awareness of detection strategies.

### Lab 11: Binary Analysis and Reverse Engineering with Ghidra
- Reverse engineered a compiled binary using Ghidra.
- Analyzed assembly instructions and program logic.
- Patched binaries to bypass security checks.
- Demonstrated techniques used in malware research and software security analysis.

## Assignments

### Assignment 1: Cryptography and Security Fundamentals
- Analyzed security scenarios involving encryption, hashing, and key usage.
- Evaluated confidentiality, integrity, authentication, and non-repudiation guarantees.
- Examined how minor changes in cryptographic operations affect bit-level representations.
- Identified subtle design flaws and incorrect security assumptions.
- Compared cryptographic primitives through logical and analytical reasoning.

### Assignment 2: Network Security and IDS Analysis
- Analyzed firewall behavior and port-based filtering rules.
- Evaluated intrusion detection system performance using true-positive and false-positive rates.
- Computed effective detection probabilities under realistic traffic distributions.
- Assessed claims related to signature-based and integrity-based detection methods.
- Applied quantitative reasoning to evaluate layered security defenses.
